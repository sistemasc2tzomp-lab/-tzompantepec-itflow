# Contribuir a ITFlow

¡Gracias por tu interés en contribuir al sistema ITFlow del H. Ayuntamiento de Tzompantepec!

## 📋 Cómo Contribuir

### 1. Fork y Clone
```bash
# Fork el repositorio
git clone https://github.com/TU_USUARIO/tzompantepec-itflow.git
cd tzompantepec-itflow
```

### 2. Crea una Rama
```bash
git checkout -b feature/nueva-caracteristica
# o
git checkout -b fix/correo-bug
```

### 3. Haz tus Cambios
- Sigue las convenciones de código existentes
- Comenta tu código cuando sea necesario
- Prueba tus cambios

### 4. Commite tus Cambios
```bash
git add .
git commit -m "feat: agregar nueva funcionalidad de exportación"
# o
git commit -m "fix: corregir error en login de usuarios"
```

### 5. Push y Pull Request
```bash
git push origin feature/nueva-caracteristica
# Luego crea un Pull Request en GitHub
```

## 🎯 Tipos de Contribuciones

### ✨ Características
- Nuevas funcionalidades
- Mejoras en la UX/UI
- Optimizaciones de rendimiento

### 🐛 Bugs
- Reporte de errores
- Correcciones de problemas existentes

### 📚 Documentación
- Mejorar README
- Agregar comentarios al código
- Crear guías de uso

### 🎨 Diseño
- Mejoras visuales
- Responsive design
- Accesibilidad

## 📝 Convenciones de Commits

Usa el formato de [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` nueva característica
- `fix:` corrección de bug
- `docs:` cambios en documentación
- `style:` cambios en formato/estilo
- `refactor:` refactorización
- `test:` agregar/modificar tests
- `chore:` cambios en herramientas/build

## 🧏‍♂️ Código de Conducta

- Sé respetuoso con otros contribuyentes
- Acepta feedback constructivo
- Ayuda a nuevos contribuyentes
- Mantén el código limpio y documentado

## 🚀 Despliegue

Los cambios se despliegan automáticamente a Vercel cuando se hace merge a la rama `main`.
